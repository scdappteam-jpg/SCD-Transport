# คู่มือใช้งานระบบ SmartLogistics

ระบบ SmartLogistics เป็นระบบติดตามงานขนส่งและคลังสินค้า สำหรับงาน Pickup, WH3 Inbound, Outbound Terminal, Billing, Dashboard และการบริหารทีม โดยมีทั้งหน้า Web App สำหรับคอมพิวเตอร์ และ Mobile App สำหรับพนักงาน/คนขับ

## 1. การเปิดระบบ

เปิดไฟล์ `Start-SmartLogistics.cmd` หรือพิมพ์ใน Terminal:

```powershell
.\Start-SmartLogistics.cmd
```

จากนั้นเปิดลิงก์:

- Web App สำหรับคอม: `http://localhost:3000/web`
- Mobile App สำหรับมือถือ/พนักงาน: `http://localhost:3000/mobile`

หมายเหตุ: ต้องเปิด Terminal ค้างไว้ระหว่างใช้งานระบบ

## 2. ภาพรวมระบบที่ทำได้

ระบบรองรับงานหลักดังนี้:

- เปิดใบงานรับสินค้าและออก Cargo Pickup Form
- Import งานจากไฟล์ CSV / Flight Feed
- ตรวจงานซ้ำจากเลข House Number
- แจ้งผล Import ว่างานใหม่กี่งาน งานซ้ำกี่งาน และมีงานเปลี่ยนแปลงกี่งาน
- Preview ไฟล์ก่อน Import และต้องกดยืนยันก่อนนำข้อมูลเข้า
- เก็บประวัติไฟล์ Import ล่าสุด 5 ไฟล์
- มอบหมายงานให้คนขับตามทะเบียนรถ
- คนขับเห็นเฉพาะงานของตัวเองใน Mobile App
- เช็คอิน / เช็คเอาต์ พร้อมเวลาและ GPS
- แนบรูปสินค้า รูปใบ Cargo และเอกสารได้หลายไฟล์
- รับสินค้าเข้า WH3 ด้วยการสแกน House และ Location
- ย้าย Location สินค้าได้
- Outbound ไป Terminal พร้อมเอกสาร Cargo Transfer, Permit, Lithium
- ตรวจ X-Ray, Re-X-Ray, Hold และ Loading Detail
- รองรับ Process สินค้าตีกลับ
- ออกใบวางบิลแบบรายงานหรือแบบกลุ่ม
- Preview ใบวางบิล แนบเอกสาร และเตรียมส่งอีเมล
- Dashboard ดูภาพรวมย้อนหลัง 10 วัน หรือเลือกช่วงวันที่เอง
- Export รายงานเป็น PDF, Excel และรูปภาพหน้าจอ
- Calendar ดูภาพรวมรายเดือน และคลิกงานเพื่อไปหน้า Tracking
- Alerts Center รวมแจ้งเตือนงานใหม่/งานเปลี่ยน/เหตุการณ์สำคัญ
- Staff Management เพิ่มพนักงานทุกฝ่ายและดู KPI

## 3. เมนูหลักใน Web App

### Dashboard

ใช้ดูภาพรวมงานทั้งหมด เช่น:

- จำนวนงานรับเข้า
- จำนวนงานส่งออก
- งานรอดำเนินการ
- งานเสี่ยงเกินเวลา
- กราฟปริมาณงานรายวัน
- สัดส่วนสถานะงาน
- ตารางงานล่าสุด

สามารถกรองข้อมูลด้วย:

- วันที่เริ่มต้น
- วันที่สิ้นสุด
- สถานะงาน

สามารถ Export ได้:

- PDF
- Excel
- รูปภาพรายงาน

ก่อน Export ระบบจะแสดงหน้าต่างให้ยืนยันช่วงวันที่และสถานะก่อน

### Orders / Tracking

ใช้ติดตามงานราย House Number

สิ่งที่ดูได้:

- รายการงานทั้งหมด
- สถานะปัจจุบัน
- Timeline การทำงาน
- Cargo Form
- รายละเอียดสินค้า
- คนขับ
- ปลายทาง
- จำนวนสินค้า
- เอกสารแนบ

ปุ่ม Today, Pending, Overdue และ All ใช้กรองรายการงานตามสถานะ

### Calendar

ใช้ดูงานแบบรายเดือน

สิ่งที่ทำได้:

- ดูงานตามวันที่
- เห็นแท็กสีตามสถานะ
- กดงานในปฏิทินแล้วไปหน้า Tracking ได้ทันที
- เลื่อนไปเดือนก่อนหน้า/เดือนถัดไป
- กลับเดือนปัจจุบัน

สถานะใน Calendar แบ่งสี เช่น:

- รอดำเนินการ
- รับเข้า
- ส่งออก
- เสร็จสิ้น / วางบิล
- เกินเวลา

### Staff

ใช้จัดการพนักงานทุกฝ่าย

รองรับบทบาท:

- Driver
- WH Staff
- Terminal
- Billing
- Admin
- Executive

สิ่งที่ทำได้:

- เพิ่มพนักงาน
- แก้ไขข้อมูลพนักงาน
- ระบุทะเบียนรถสำหรับคนขับ
- ดูจำนวนงาน
- ดูงานที่เสร็จแล้ว
- ดูเวลาเฉลี่ย
- ดู KPI
- ดูกำลังคนแยกตามฝ่าย

### LINE & Mobile Ops

เป็นหน้าจอจำลอง Mobile App / LINE-style workflow

ใช้กับ:

- คนขับ
- พนักงาน WH3
- พนักงาน Terminal
- บัญชี

หน้าจอจะแยกเมนูการทำงานตามบทบาท เพื่อไม่ให้ผู้ใช้เห็นงานที่ไม่เกี่ยวข้อง

### Admin

ใช้สำหรับแอดมินเปิดงานและ Import งาน

สิ่งที่ทำได้:

- เปิดใบงานใหม่
- กรอกข้อมูลล่วงหน้า
- เลือกลูกค้า
- เลือกคนขับ/ทะเบียนรถ
- เพิ่มหลาย House ในใบเดียว
- Preview Cargo Pickup Form
- Print Cargo Pickup Form
- Import CSV
- Preview ไฟล์ก่อน Import
- ยืนยัน Import
- ดูงานรอเปิด
- จัดกลุ่มงานตามลูกค้า/สถานที่/รอบงาน
- มอบหมายงานให้คนขับ
- ดูประวัติไฟล์ Import ล่าสุด 5 ไฟล์

### Alerts

ใช้ดูแจ้งเตือนแยกจากหน้า Admin

ตัวอย่าง Alert:

- งานใหม่จากไฟล์ Import
- งานมีการเปลี่ยน Flight
- ปลายทางเปลี่ยน
- จำนวนสินค้าเปลี่ยน
- ยังไม่ออก Cargo Form
- Re-X-Ray / Rescan
- เหตุการณ์สำคัญจากระบบ

### Settings

ใช้ดูค่าระบบ เช่น:

- API Server
- Database
- Storage
- LINE Alert
- Role Permission

## 4. Module 1: Pickup รับสินค้าจากคลังลูกค้า

### กรณีแอดมินเตรียมใบงานให้

1. แอดมินเปิดหน้า Admin
2. กรอกข้อมูลลูกค้า วันรับสินค้า สถานที่รับสินค้า คนขับ ทะเบียนรถ
3. เพิ่ม House Number และปลายทาง
4. Preview Cargo Pickup Form
5. กดเปิดใบงาน
6. Print Cargo Pickup Form
7. งานจะไปแสดงใน Mobile App ของคนขับ

### กรณีคนขับเขียนใบเอง

1. แอดมินเปิดใบงานเปล่าให้คนขับ
2. คนขับไปหน้างานและเขียน Cargo Pickup Form เอง
3. คนขับกดเช็คอิน
4. ถ่ายรูปสินค้า
5. เสร็จงานแล้วถ่ายรูปใบ Cargo ที่เซ็นแล้ว
6. กดจบงาน / Check-out

### ข้อมูลที่ระบบเก็บ

- เวลาเริ่มงาน
- เวลาจบงาน
- GPS
- รูปสินค้า
- รูปใบ Cargo
- จำนวนงานต่อคนขับ
- ระยะเวลาที่ใช้ต่อใบงาน

## 5. Module 2: WH3 Inbound

ใช้สำหรับรับสินค้าเข้าคลัง WH3

ขั้นตอนหลัก:

1. รับ Cargo Pickup Form จากคนขับ
2. ตรวจ Invoice / PO
3. ถ้าไม่พบเอกสาร ตั้งสถานะรอ EI Confirm
4. เมื่อเอกสารพร้อม เปิดงานรับเข้า
5. สแกน House Barcode
6. ตรวจ Flight / จำนวน / ปลายทาง
7. แยก Track คู่ / Track เดี่ยว
8. ถ้าเป็น WD หรือสินค้าพิเศษ ทำ Checklist และติด Sticker
9. ระบุ Location
10. สแกน House + Location
11. ปิดงานรับเข้า

ระบบรองรับ:

- ยิง Barcode ด้วยเครื่องสแกน
- กรอก House เอง
- ย้าย Location
- เปลี่ยนสถานที่จัดเก็บ
- ดู Location ว่าง/ไม่ว่าง

## 6. Module 3: Outbound Terminal

ใช้สำหรับเบิกสินค้าออกจาก WH3 ไป Terminal

ระบบแบ่งเป็น 2 ส่วน:

### ส่วนแอดมินเตรียมงาน

- Import งานขาออกจากไฟล์ CSV
- เลือกงานที่จะออก
- เตรียม Cargo Transfer Form
- เตรียม Permit
- เตรียมเอกสาร Lithium ถ้ามี
- จองรถ AOT / Terminal

### ส่วนพนักงานปฏิบัติการ

1. เห็น Location ก่อนเพื่อไปหยิบสินค้า
2. เปิดกล้องหรือยิง Barcode House
3. เมื่อเจอสินค้าแล้วจึงแสดงรายละเอียด
4. Scan เข้าระบบ EI
5. เคลียร์ Location เป็น Available
6. ให้ EI เซ็นเอกสาร
7. ถ่ายรูปเอกสารที่เซ็นแล้ว
8. โหลดสินค้าไป Loading Bay
9. สแกนเช็คกับ Manifest
10. ถ่ายรูปก่อนปิดท้ายรถ
11. คนขับส่ง Terminal
12. เช็คอิน GPS เมื่อถึง Terminal
13. Terminal ตรวจเอกสาร
14. ชั่งน้ำหนัก / วัด Dimension
15. X-Ray
16. Packing / Loading Detail
17. ปิดงานเพื่อส่งต่อ Billing

ระบบรองรับ:

- Cargo Transfer Form จากระบบ
- Permit / Lithium document
- X-Ray ผ่าน
- Re-X-Ray
- Hold
- Requires Rescan Alert
- 4-hour flight warning
- สินค้าตีกลับอีก 1 Process

## 7. Module 4: Billing & Dashboard

ใช้สำหรับบัญชีวางบิลหลังงาน Terminal ปิดและมี Loading Detail

สิ่งที่ทำได้:

- ดูงาน Ready for Billing
- ตรวจเอกสารก่อนวางบิล
- ใบชั่งน้ำหนัก
- Cargo Transfer Form
- Loading Detail
- เอกสารหน้างาน
- สร้าง Draft Invoice
- Preview Invoice
- Print Invoice
- แนบเอกสารสำหรับส่งอีเมล
- ส่งอีเมลวางบิล
- Mark Billed
- นับ Due Date ตาม Credit Term

ระบบรองรับการวางบิลแบบกลุ่ม:

- รวมหลาย House ในบิลเดียว
- รวมตามลูกค้า
- รวมตามเดือน
- รวมสิ้นเดือน
- ดูยอดวางบิลแล้ว / ยังไม่วางบิล

## 8. การ Import ไฟล์ CSV

เมนู: Admin > Flight Feed Import

ขั้นตอน:

1. กด Choose File
2. เลือกไฟล์ CSV
3. ระบบแสดง Preview ไฟล์
4. กดตรวจสอบและยืนยัน Import
5. ระบบเปิด Popup ให้ตรวจข้อมูล
6. กดยืนยัน Import
7. ระบบสรุปผล

ระบบจะแจ้ง:

- มีทั้งหมดกี่แถว
- งานใหม่กี่งาน
- งานซ้ำกี่งาน
- งานเปลี่ยนแปลงกี่งาน
- House ไม่ซ้ำกี่งาน
- ยังไม่ออก Cargo กี่งาน

ระบบใช้ House Number ในการตัดซ้ำ หาก House เดิมถูก Import ซ้ำ ระบบจะไม่สร้างงานใหม่ แต่จะนับเป็นข้อมูลซ้ำหรืออัปเดตหากข้อมูลเปลี่ยน

## 9. Mobile App

เปิดที่:

`http://localhost:3000/mobile`

ผู้ใช้ต้องเลือกบัญชี/บทบาทก่อนใช้งาน

หน้าจอมือถือออกแบบให้เหมือนขั้นตอนทำงาน:

- เห็นเฉพาะเมนูที่เกี่ยวข้องกับบทบาท
- ต้องเริ่มจากการเลือกงานหรือสแกนก่อน
- รายละเอียดจะแสดงหลังจากเจองาน
- แต่ละขั้นมีปุ่มเริ่ม / จบ เพื่อเก็บเวลา KPI
- รองรับถ่ายรูปและแนบเอกสาร

## 10. KPI และรายงาน

ระบบคำนวณได้:

- จำนวนงานต่อคน
- งานค้าง
- งานเสร็จ
- เวลาเฉลี่ย
- KPI รายบุคคล
- KPI รายฝ่าย
- งานค้างตามฝ่าย
- จุดที่เกิดปัญหาบ่อย
- งานเกินเวลา
- Billing pending
- Billed amount
- Pending amount

## 11. ข้อควรจำ

- หลังแก้ระบบหรืออัปเดตไฟล์ ให้กด `Ctrl + F5`
- ถ้าเปิดเว็บไม่ได้ ให้ตรวจว่า Server รันอยู่หรือไม่
- ถ้า Port 3000 ถูกใช้อยู่ ให้ปิด Server เก่าก่อน
- Import ต้องใช้ไฟล์ CSV
- House Number คือข้อมูลหลักในการตรวจซ้ำ
- งานมือถือและเว็บใช้ข้อมูลชุดเดียวกัน
